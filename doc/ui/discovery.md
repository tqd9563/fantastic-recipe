# 发现（Discovery）

进入「发现」页，首先映入眼帘的是两个醒目的大卡片/板块，引导用户选择探索路径。  
布局：50% - 50% 左右分栏布局（移动端为上下排列）。

- 左侧：全网搜罗 (Web Search)

  - 视觉：背景可以是精美的食材拼接图，蒙上一层半透明遮罩。
  - 文案：主标题“全网搜罗”，副标题“汇集下厨房、小红书的热门美味”。
  - 核心元素：一个显眼的放大镜图标 🔍。
  - 交互：鼠标悬停或点击时，背景图轻微放大，产生“进入”的沉浸感。

- 右侧：AI 创意厨房 (AI Chef)

  - 视觉：背景使用科技感或极简风格的线条插画（如一个机器人厨师）。
  - 文案：主标题“AI 创意厨房”，副标题“剩菜大变身 & 灵感生成器”。
  - 核心元素：一个闪烁的魔法棒图标 🪄 或 机器人图标 🤖。
  - 交互：？


## 1. 全网搜罗 (Web Search)

**核心：突出“海量资源”和“筛选效率”**

- 顶部搜索栏：

  - 大搜索框：居中，Placeholder 提示“输入菜名、食材（如：红烧茄子、土豆...）”。
  - 来源切换 Tab：搜索框下方提供 [综合]、[下厨房]、[小红书]、[美食杰] 等来源筛选。默认全选
  - 链接导入入口：搜索框右侧放一个小的 [粘贴链接] 按钮，点击弹窗支持直接粘贴 URL 解析。

- 内容展示区：

  - 布局：采用 瀑布流布局，因为不同来源的图片比例不一。
  - 单张卡片结构

    - 封面图：占据卡片 70% 高度，必须大且清晰。
    - 标题：双行截断，粗体。
    - 来源角标：卡片左下角显示来源 Logo（如小红书图标）。
    - 元数据：右下角显示 ❤️ 收藏数/点赞数（证明这道菜很火）。

  - 悬停交互

    - 鼠标悬停在卡片上时，出现一个醒目的 [+ 收录] 按钮。
    - 点击 [+ 收录] 后，自动抓取该网页内容，弹出一个简化版的“编辑食谱”弹窗（预填好抓取到的信息），用户可进行编辑，确认无误后保存进入食谱库中


- 底部加载：无限滚动 (Infinite Scroll)，无需分页。

## 2. AI 创意厨房 (AI Chef)

**核心：突出“对话感”和“惊喜感”**  
分步引导式交互，不要一下子把所有表单丢给用户，分两步走。

- Step1：需求输入 (Input)

  - 模式选择 Tab

    - 口味探索（默认）

      - 偏好选择器：提供一组可点击的胶囊按钮（酸甜、麻辣、清淡、红烧...）
      - 主食材选择（可选项，默认全部）：下拉菜单选择（猪肉、鸡肉、豆腐...）

    - 冰箱大扫除

      - 食材输入框：支持 Tag 输入（输入“鸡蛋”回车，变成一个胶囊 Tag）
      - 提示文案：“告诉我你冰箱里还剩下什么？（如：半个洋葱、两个番茄）”


  - 生成按钮：一个巨大的、带渐变流光效果的 [✨ 开始生成] 按钮。

- Step2：灵感生成结果

  - 生成动效（加载状态）：显示一个正在炒菜的动画，配上文案“AI 正在翻阅米其林菜谱...”，底部可以加一个进度条加载
  - 结果展示：

    - AI 通常生成 1-3 个方案。建议使用 「横向大卡片轮播」形式展示
    - 卡片内容：

      - 菜名（AI自动生成），如金风玉露一相逢 - 洋葱炒蛋进阶版
      - 效果图（如果技术允许，调用 DALL-E 生成；如果不行，用通用占位图）
      - 推荐理由：这道菜完美消耗了你的洋葱，且符合你喜欢的酸甜口味。”
      - 主要食材及步骤预览

    - 卡片底部操作栏：

      - ❌不喜欢，重试：重新生成
      - ❤️看着不错，收录：一键保存到食谱库，并标记为“从未尝试”
