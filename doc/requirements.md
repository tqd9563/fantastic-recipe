# 项目简介
这是一个主要用于个人食谱管理的网站

# 核心功能
## 收录（Collection）
支持用户收录食谱，内容包括但不限于：
- 基础信息
  - 名称
  - 描述
  - 封面图片URL
- 核心信息
  - 食材列表，每份食材信息包括
    - 食材名字
    - 分量
    - 种类
  - 配料表
  - 预计耗时
  - 难度（1-5星）
  - 评分（1-5星）
- 标签信息
  - 主分类（大荤/小荤/素菜/汤）
  - 子分类（猪肉/鸡肉/叶菜/根茎菜/...）
  - 口味（酸/甜/辣/咸/清蒸/爆炒/红烧/炖煮/凉拌...）
  - 末次食用时间（每次将菜谱加入“日历计划”并标记完成时，自动更新该字段。）
  - 总制作次数
  - 熟练度（从未尝试/新手/熟练/精通，其中”从未尝试“对应了通过「发现」功能导入的食谱，需要特别标注）
- 烹饪步骤

## 检索
基于食谱收录的功能，支持以下检索功能：
- 根据指定食谱名称搜索（支持模糊匹配）
- 根据特定的食材名称，搜索所有用到这个食材的食谱

## 计划（Planing）

**核心目标**：帮助用户生成未来几天的食谱安排，根据已有的食谱结合各种策略进行推荐。

- 日历系统：支持按天更新当日食谱
  - 无需区分早中晚，但需要支持一天加入多个食谱（多个菜）
- 一键计划：基于选定的日期范围，帮我自动安排每天的食谱
  - 系统自动根据推荐算法逻辑，为我进行填充。具体的算法逻辑可参见下方# 智能推荐算法逻辑这一节
  - 默认配置是每天两道菜，一荤一素，可支持用户自定义选项（如想要一荤两素）
- 采购清单：基于选定的日期范围，和计划的食谱，自动汇总所需的食材列表

## 发现（Discovery）

**核心目标**：帮助用户拓展食谱库，寻找灵感，发现未曾尝试过的新菜品。

- 外部探索
  - 全网搜罗：用户输入关键词（如“茄子”），聚合展示网络上热门的新做法，而非仅仅搜索本地库。
    - 搜索出来的结果，支持用户挑选自己满意的食谱后，一键加入食谱库
  - 链接导入：看到小红书/下厨房的好菜谱，支持粘贴链接，系统自动解析并结构化存入你的食谱库。
- 创意厨房：
  - AI定制：用户输入一些冰箱里剩余的食材组合，例如金针菇 + 萝卜，AI为你生成一道没有做过的菜
  - 口味拓展：根据口味喜好，如酸甜，猪肉，等等，推荐一道从未做过的菜。

> 注意：通过「发现」模块加入食谱库中的食谱，其熟练度为”从未尝试“

## 统计(Stats)
可视化地展示以下数据：
- 最常吃的菜TopN
- 偏好雷达图，例如口味偏好、肉类/菜类偏好等
- 做饭频率热力图，类似Github Contribution，鼓励自己多做饭
- 其他

# 智能推荐算法逻辑
综合多个因子综合加权后的推荐
- 冷却期：排除最近N天吃过的菜谱（不重复吃）
- 时间因素：周一到周五推荐相对简单的（耗时较少），周六日可适当推荐复杂的食谱
- 特殊的去库存机制：用户输入当前冰箱里的剩余食材，则算法会优先匹配可使用这些食材的食谱
- 应季性：根据当前季节，优先选择应季的蔬菜
- 评分权重：自己打分高的菜优先（爱吃的优先）
- 随机性：引入少量随机因子，防止推荐列表固化
